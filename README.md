## 🛠️ 3. Collaborative Development Workflow

To prevent data corruption and codebase collisions on the mainframe, all development teams must strictly adhere to the **GitHub Flow** architecture:

1.  **Isolate Work:** Never commit code directly to the `main` production trunk. 
2.  **Branch Creation:** Instantiated branches must follow strict corporate nomenclature:
    ```bash
    git checkout -b feature/yourEmployeeID-feature-name
    ```
3.  **The Commit Audit:** Stage your micro-adjustments with clear, professional logs:
    ```bash
    git commit -m "Docs: Update system interface parameters for employee check-in"
    ```
4.  **The QA Gate:** Push your feature branch upstream and open a **Pull Request (PR)**. Assign a Cybersecurity Analyst or QA teammate to audit your logic. 
5.  **The PM Review:** Once a peer stamps the PR with an approval comment (`LGTM`), the Project Manager will merge the branch and close the tracked issue ticket.
